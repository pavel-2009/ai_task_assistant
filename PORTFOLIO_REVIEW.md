# Ревью проекта для портфолио: полный список исправлений

Ниже — список того, что нужно исправить в проекте, чтобы его можно было уверенно показывать в портфолио как production-like backend/ML API.

## 1) Безопасность и секреты

- Удалить реальный/примерный API-ключ из `app/core/config.py` и читать `LLM_API_KEY` только из `.env`.
- Ротировать (перевыпустить) ключ, который уже попал в репозиторий.
- Добавить `.env.example` без секретов и обновить README с безопасной инструкцией запуска.
- Добавить pre-commit/CI проверку на утечки секретов (например, gitleaks/trufflehog).

## 2) Критические runtime-баги

- Исправить вызов `semantic_search_service.delete(...)` в `app/routers/tasks.py`: сейчас в вызов передаётся `session`, а метод в `SemanticSearchService` такой параметр не принимает.
- Исправить путь `@router.get("/tasks/{task_id}/tags_status")` в роутере с префиксом `/tasks`: сейчас получается некорректный URL `/tasks/tasks/{task_id}/tags_status`.
- Исправить вызов `semantic_search_service.index_async(...)` в `app/ml/nlp/tasks.py`: такого метода в `SemanticSearchService` нет.
- Исправить вставку в таблицу `Text` в `app/ml/nlp/tasks.py`: используется поле `item_id`, которого нет в ORM-модели (`Text` содержит `text_id`).
- Привести к единому типу идентификаторы при индексировании/удалении вектора (`int`/`str`) во всех местах, где используются `VectorDB` и `RecSysVectorDB`.
- Исправить `RecSysVectorDB`: методы используют `self._lock`, но лок не инициализируется в `__init__`.
- Исправить асинхронность `RecSysVectorDB`: сейчас есть смешение sync/async Redis-API и вызовы без `await`.
- Исправить тип/формат сериализации FAISS индекса в `RecSysVectorDB` (сейчас сериализация/десериализация используется непоследовательно).
- Исправить ошибку в `check_avatar_drift`: в `detector.add_embedding([embedding])` передаётся лишний уровень вложенности.

## 3) Логика API и контракты

- Для всех роутов явно указать `response_model`, чтобы Swagger и реальные ответы совпадали.
- В `create_task` вернуть DTO (`TaskGet`), а не ORM-объект.
- Удалить неиспользуемые параметры `request: Request = None` в эндпоинтах, где они не применяются.
- В endpoint-ах статусов Celery убрать проверки `if not celery_task` (они невалидны для `AsyncResult`) и проверять существование/статус корректно.
- Привести HTTP-методы к семантически корректным (`GET` для чтения статуса/результатов, `POST` для создания задач обработки).
- Для `DELETE /tasks/{task_id}` убрать тело ответа и гарантировать корректный `204 No Content`.
- Везде унифицировать формат ошибок: единая структура `{"detail": ..., "code": ...}` + единые коды.

## 4) Качество ML/NLP логики

- В классификации изображений заменить игрушечный `idx_to_class = {cat,dog,house}` на реальные классы модели и хранить mapping рядом с checkpoint.
- Добавить валидацию соответствия checkpoint и архитектуры модели перед загрузкой.
- Везде, где есть вероятность нулевой нормы вектора, добавить safe-normalization (проверка перед делением).
- Для рексиса перестроить объединение текстового и image embedding: сейчас размерности и типы тензоров/массивов смешиваются хрупко.
- В `SemanticSearchService` и `VectorDB` унифицировать поля результата (`task_id`/`text_id`) и использовать одну схему ответа.
- В `RAGService` добавить guardrails на пустые/мусорные ответы LLM, таймауты и fallback с диагностикой причины.
- Вынести prompt-шаблоны RAG в отдельные файлы/конфиг вместо hardcode в коде.

## 5) Работа с файлами и путями

- Убрать жестко зашитые абсолютные пути вида `/app/...`, перейти на конфигурируемые директории (`AVATARS_DIR`, `SEGMENTS_DIR`, `CHECKPOINTS_DIR`).
- Проверять и нормализовывать пути файлов до чтения/записи, чтобы исключить path traversal и некорректные ссылки.
- Добавить очистку старых артефактов (`avatars`, `segments`, временные файлы) и lifecycle-политику хранения.

## 6) Миграции и БД

- Добавить реальные ревизии Alembic в `alembic/versions` (сейчас в репозитории нет миграций схемы).
- Добавить FK связи (`Task.author_id -> User.id`, `Interaction.user_id/task_id`) и индексы под частые запросы.
- Для `Interaction.created_at` добавить серверный default (`func.now()`), чтобы не зависеть от ручного заполнения.
- Добавить ограничения уникальности/целостности там, где логика их предполагает.

## 7) Тесты

- Починить запуск тестов из чистого окружения: сейчас `pytest` падает уже на стадии import.
- Разделить unit/integration/e2e тесты и добавить маркировки (`@pytest.mark.unit`, `integration`, `slow`).
- Добавить тесты на проблемные места:
  - удаление задачи + удаление из векторной БД,
  - Celery статус-эндпоинты,
  - обновление/переиндексация текста,
  - RecSysVectorDB add/update/delete/search,
  - корректность RAG cache behavior.
- Добавить smoke-тест Docker-компоновки (`api + redis + celery`) в CI.

## 8) CI/CD и стандарты качества

- Включить проверку импортов/мертвого кода (`ruff F401/F841`, autoflake/pycln).

- Разделить зависимости на base/dev/ml-extra и сделать воспроизводимую установку.

## 11) Наблюдаемость и эксплуатация

- Добавить структурированные логи (json) с request_id/task_id/user_id.
- Для Celery добавить явные retry-policy, backoff и DLQ/обработку "ядовитых" задач.
- Добавить метрики бизнес-уровня: время ответа RAG, hit-rate cache, доля ошибок инференса, размер индекса.

- Прогон автотестов без падений на import/runtime.
